# 本地前端分享器

这是一个用于管理本地 React + Vite 项目的分享工具。

它现在支持两种使用方式：

1. 本地 Web 控制台
2. Windows 桌面程序

核心目标是让同事安装后直接打开程序，选择自己的前端项目，一键生成：

- 本地预览地址
- 公网分享地址

## 当前分享逻辑

为了避免 `vite dev + HMR + tunnel` 导致的公网白屏，当前分享链路固定为：

1. `npm run build`
2. `npm run preview -- --host 0.0.0.0`
3. `localtunnel` 生成公网地址

也就是说，公网访问走的是“构建后预览”，不是直接暴露开发态页面。

## 已完成的关键能力

- 添加并保存多个本地 React + Vite 项目
- 启动、停止、重启单个项目分享
- 展示本地地址和公网地址
- 自动分配分享端口，默认从 `4173` 开始
- `localtunnel` 已改为项目内依赖，不再依赖运行时 `npx` 临时拉取
- 新增 Electron 桌面外壳
- 打开桌面程序时自动拉起内部服务
- 关闭桌面程序时自动停止内部服务

## 开发环境运行

安装依赖：

```bash
npm install
```

启动 Web 版控制台：

```bash
npm run dev
```

打开：

```text
http://localhost:3000
```

启动桌面版：

```bash
npm run desktop
```

## 打包 Windows 安装包

先确保依赖已安装，然后执行：

```bash
npm run dist:win
```

打包产物默认输出到：

```text
release/
```

## 面向同事的使用方式

未来给同事使用时，目标流程是：

1. 安装桌面程序
2. 打开程序
3. 选择本地 React + Vite 项目目录
4. 点击启动分享
5. 程序自动完成构建、预览和公网地址生成

正常使用时，不需要他们手动打开 PowerShell，也不需要手动执行命令。

## 当前约束

- 目标项目仍然限定为 React + Vite
- 公网分享依赖 `localtunnel.me` 网络可达
- `localtunnel` 域名是动态分配的，不保证固定
- 首次打开某些 `*.loca.lt` 地址时，浏览器可能会先显示安全提示页

## 已知事项
